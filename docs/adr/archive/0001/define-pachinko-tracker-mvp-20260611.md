# 0001 - パチンコ収支管理アプリの MVP 機能定義

## Status

Proposed

## Context

自分用の個人アプリとして、パチンコ実戦の投資・回収・収支を素早く記録し、あとからホール別・機種別・日別/月別に勝ち負けの傾向を確認したい。

スマホでの現場入力を最優先にし、PC は一覧・分析が見やすければ十分とする。初期実装ではログインやクラウド同期を持たず、端末内保存で素早く使い始められることを重視する。

## Options Considered

### 1. 日別まとめを主軸にする

- **メリット**: 入力項目が少なく、ざっくり管理しやすい。
- **デメリット**: 1 日に複数台打った場合、機種別・台別の振り返りが弱くなる。

### 2. 台ごとの実戦ログを主軸にする

- **メリット**: 機種別・ホール別・日別/月別の分析がしやすい。将来の機種データ拡張とも相性がよい。
- **デメリット**: 入力単位が細かくなるため、入力導線を軽くする必要がある。

## Decision

MVP は「台ごとの実戦ログ」を主軸にする。

実戦ログの必須項目は `日付・ホール名・機種・投資・回収` とし、収支は自動計算する。任意項目として `開始時刻・終了時刻・交換率・メモ・タグ` を持たせる。

機種は自由入力ではなく、開発時に seed データとして事前登録した機種一覧から選択する。MVP の機種データは `機種名・メーカー・カテゴリ・メモ` 程度に留め、将来的に `演出・スペック・狙い目・注意点` などへ拡張できるようにする。

ホールは詳細マスタ化せず、`ホール名` のみ扱う。必要に応じて過去入力候補を出す程度にする。

集計は `日別・月別・ホール別・機種別` を対象にし、指標は `収支・勝率・総投資・総回収・平均収支` とする。グラフは `月別推移` と `カテゴリ別ランキング` を MVP に含める。

初期画面は収支ログ画面を中心にし、上部に `今月の収支サマリー`、下に `実戦ログ一覧` と `追加ボタン` を置く。`分析画面` と `機種一覧` は別タブにする。

データ保存はまず端末内保存とし、ログインやクラウド同期は不要とする。ただし将来的に CSV エクスポート/インポートやクラウド同期を追加できる余地は残す。

スマホ利用を最優先にする。ホールや移動中に片手で入力する前提で、Web アプリとして作る場合もモバイルファーストにする。PC は一覧・分析が見やすければ十分とする。

## Consequences

実戦後すぐに記録でき、あとから自分の勝ち負け傾向を実用的に振り返れる。台ごとの実戦ログを主軸にすることで、日別・月別・ホール別・機種別の集計を同じデータから生成できる。

機種を seed データとして扱うことで、将来的な機種情報データベース化にもつなげやすい。一方で、初期リリースではユーザーが自由に機種を追加・編集する機能は持たない。機種追加は開発作業として行う。

端末内保存のため、初期実装は軽くできるが、複数端末利用やバックアップは後続課題とする。
