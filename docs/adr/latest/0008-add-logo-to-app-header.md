# 0008 - 共通ヘッダーにロゴを表示する

## Status

Proposed

## Context

サイト用ロゴ `src/img/p-web-logo.png` を作成済みである。現在の共通ヘッダーはテキストのみで、ブランドの視認性を高める余地がある。

アプリはモバイル幅中心のレイアウトで、ログ、分析、機種などの主要画面を共通シェル内で表示している。そのため、既存の画面密度を崩さず、全主要画面で一貫してロゴを表示する必要がある。

## Options Considered

### 1. 共通ヘッダー左に小さく表示し、右に既存タイトルを置く

- **メリット**: 全画面で自然にブランドを示せる。既存レイアウトへの影響が小さい。
- **デメリット**: ロゴの表示サイズは控えめになり、ビジュアル訴求は限定的。

### 2. ログ一覧などトップ画面に大きく表示する

- **メリット**: 初回表示でロゴの印象を強く出せる。
- **デメリット**: 実用画面の情報密度が下がり、他画面との一貫性も弱くなる。

## Decision

共通ヘッダー左にロゴを小さく表示し、右に既存の `Pachinko Wallet Tracker` と `パチンコ収支管理` を維持する。

この配置により、ロゴを全主要画面で一貫して見せながら、既存のモバイル向け操作画面の密度を保つ。

## Consequences

ブランド認知が上がり、全主要画面で一貫した見え方になる。

ヘッダーの高さが少し増える可能性があるため、画像サイズと余白は固定し、モバイルでの表示崩れを避ける。ロゴ画像の差し替え時も、同じ縦横比と表示サイズを前提に確認する必要がある。
