# 0010 - 台ごとの補正済み平均回転数を記録・集計する

## Status

Proposed

## Context

既存の実戦ログには、投資額ごとの回転数メモがある。現在は記録・編集・一覧表示までを対象としており、台ごとの平均回転数分析は未実装である。

ユーザーは「その台がどれくらい回るか」を後から振り返りたい。ただし、現金投資額だけを分母にして `1,000円あたり回転数` を計算すると、当たり後の持ち玉で通常回転を回した分が反映されず、台の実力評価が歪む可能性がある。

また、同じ機種でもホールや台番号が違えば回転状況は別物になりやすい。そのため、機種単位だけではなく、ホールと台識別を含めた集計単位を扱える設計が必要になる。

## Options Considered

### 1. 現金投資額と回転数メモだけで `1,000円あたり回転数` を出す

- **メリット**: 既存データだけで実装できる。入力項目が増えない。
- **デメリット**: 当たり後に通常回転へ使った出玉を考慮できず、台の実力評価が歪みやすい。

### 2. 最終回収額を使って補正する

- **メリット**: 既存の回収入力を流用できる。
- **デメリット**: 最終回収額は残った出玉や換金結果であり、通常回転へ再投資した出玉とは一致しない。

### 3. 通常回転へ使った出玉を玉数で任意入力し、貸玉レートで補正する

- **メリット**: 持ち玉で回した通常回転分を投資相当に換算でき、台の平均回転数を実戦感に近づけやすい。
- **デメリット**: `ballRateYen` と通常回転へ使った出玉の入力項目が増える。

## Decision

選択肢 3 を採用する。

台ごとの平均回転数は、当たり中を除いた通常時の `1,000円あたり回転数` として扱う。分母は `現金投資額 + 通常回転へ使った出玉 * 貸玉レート` とし、分子は通常時の回転数メモ合計とする。

`exchangeRate` は既存の収支用として残し、回転率分析用に `ballRateYen` を追加する。出玉補正は、最終回収額ではなく「当たり後に通常回転へ使った出玉」を玉数で任意入力し、貸玉レートで金額換算する。

集計単位は `ホール + 機種 + 台番号/台識別` を基本とする。台番号/台識別は任意入力とし、未入力の場合は従来どおり `ホール + 機種` 単位で集計する。

## Consequences

台ごとの回転状況を、現金投資だけの単純計算より実戦感に近い形で記録・分析できる。持ち玉で通常回転を回した分も評価に含められるため、後から台の良し悪しを振り返りやすくなる。

一方で、ログフォーム、データモデル、集計ロジックは複雑になる。ユーザーが出玉再投資分を入力しない場合は、補正なしの平均回転数として扱う必要がある。

既存ログには `ballRateYen` や台識別が存在しないため、後方互換を維持しながら任意項目として導入する。将来的に機種平均やホール別の回転率ランキングへ展開しやすくなるが、当たり中回転を含めない運用や入力説明もあわせて整える必要がある。
