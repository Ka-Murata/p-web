# 0009 - アプリテーマを黒ベースに変更する

## Status

Proposed

## Context

現在のアプリは白・淡い緑を基調としている。主な利用シーンとして夜間やホール内のスマホ利用が想定され、白背景は眩しさや長時間利用時の疲れにつながる。

ブランドの緑・黄色アクセントは維持しつつ、全体を黒ベースにして視認性と雰囲気を改善したい。

## Options Considered

### 1. 現状の白ベースを維持する

- **メリット**: 変更コストがない。既存 UI の見た目を保てる。
- **デメリット**: 夜間・ホール内で眩しく、今回の目的を満たしにくい。

### 2. OS 設定に合わせてライト / ダークを切り替える

- **メリット**: ユーザー環境に合わせられる。将来的な柔軟性が高い。
- **デメリット**: 2 テーマ分の設計・実装・検証が必要になり、現時点では変更範囲が広がる。

### 3. 常時黒ベースへ変更する

- **メリット**: 主要利用シーンでの眩しさを抑えられる。単一テーマなので実装・検証範囲を抑えられる。ブランド感も強めやすい。
- **デメリット**: ライトテーマ利用者には見た目の変化が大きい。白背景前提の個別色指定を見直す必要がある。

## Decision

アプリ全体を常時黒ベースへ変更する。対象は背景、カード、下部ナビ、フォーム、ダイアログ、グラフ、状態表示を含む。

ロゴ画像は変更しない。既存の緑と黄色はアクセントとして継続する。

色は `pwt-*` デザイントークン中心に管理し、将来ライト / ダーク切り替えが必要になった場合に拡張できる余地を残す。

## Consequences

夜間・ホール内での眩しさを低減し、収支カードや入力フォームの視認性を高められる。

一方で、個別に指定されている赤・白・淡色系の表示やグラフ色は黒背景でのコントラスト確認が必要になる。

撤回コストは、トークン中心に実装すれば中程度と見なす。
