# 0006 - 実戦ログフォームに一覧へ戻る導線を追加する

## Status

Proposed

## Context

実戦ログ追加画面 `/logs/new` と編集画面 `/logs/:id/edit` では、フォーム操作を優先するため下部タブを非表示にしている。そのため、保存せずにログ一覧へ戻る明示的な導線がなく、スマホで現場入力中に「やっぱり戻る」操作がしにくい。

ADR 0001 ではスマホ利用を最優先とし、ホールや移動中に片手で入力する前提としている。フォーム画面でも、保存・削除だけでなく、一覧へ戻る基本導線が必要になる。

## Options Considered

### 1. 常に `/logs` へ戻るボタンを追加する

- **メリット**: 直接 `/logs/new` を開いた場合でも戻り先が安定する。追加画面・編集画面で挙動を揃えられる。
- **デメリット**: ブラウザ履歴上の直前画面には戻らない。

### 2. `navigate(-1)` でブラウザ履歴に戻る

- **メリット**: 直前の画面へ戻れる。
- **デメリット**: 直接アクセスや外部リンク経由では戻り先が不安定になる。

### 3. 未保存確認ダイアログ付きで戻る

- **メリット**: 入力途中の誤操作を防ぎやすい。
- **デメリット**: MVP としては状態管理と UI が少し重くなる。

## Decision

実戦ログ追加画面と編集画面の両方に「戻る」ボタンを追加する。

戻る先はブラウザ履歴ではなく、常に `/logs` とする。これにより、直接 `/logs/new` や `/logs/:id/edit` を開いた場合でもログ一覧へ戻れる。

MVP では未保存確認ダイアログは出さない。戻るボタンを押すと入力内容は保存せず破棄され、`/logs` に遷移する。未保存確認は後続改善候補とする。

## Consequences

フォーム画面で下部タブが非表示でも、保存せず一覧へ戻れるようになる。追加画面と編集画面で挙動が揃い、スマホでの操作が分かりやすくなる。

一方で、入力途中に戻ると内容は破棄される。MVP ではこのトレードオフを受け入れ、必要になったら未保存確認ダイアログを追加する。
