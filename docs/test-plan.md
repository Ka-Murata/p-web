# MVP 手動確認シナリオ

## 前提

- `npm install` 済み。
- `npm run dev` で http://localhost:5173/ を開く。
- MVP は端末内 IndexedDB に保存するため、ブラウザのサイトデータを消すとログも消える。

## 初回起動からログ追加

1. `/logs` を開き、ログなし状態と今月サマリーのゼロ表示を確認する。
2. 下部タブで `/machines` を開き、seed 機種が表示されることを確認する。
3. `/logs/new` を開き、日付、ホール名、機種、投資、回収を入力する。
4. 投資・回収を変えると収支プレビューが即時更新されることを確認する。
5. 保存し、`/logs` に戻って機種名、日付、ホール名、投資、回収、収支が表示されることを確認する。

## 編集・削除

1. `/logs` の実戦カードから「編集」を押す。
2. 既存値がフォームに入っていることを確認する。
3. 投資または回収を変更して保存し、一覧の収支が更新されることを確認する。
4. もう一度編集画面を開き、「削除」を押す。
5. 確認ダイアログで削除し、一覧から対象ログが消えることを確認する。

## 分析画面

1. 複数月、複数ホール、複数機種のログを追加する。
2. `/analytics` を開き、KPI、月別推移グラフ、ホール別ランキング、機種別ランキングが表示されることを確認する。
3. ログがない状態では分析データなし状態が表示され、画面が崩れないことを確認する。

## 機種画面

1. `/machines` を開き、機種名、メーカー、カテゴリ、メモを確認する。
2. 機種名またはメーカー名で検索し、一覧が絞り込まれることを確認する。
3. 検索入力を消すと一覧が戻ることを確認する。
4. 機種を選択し、詳細プレビューが更新されることを確認する。

## モバイル表示

1. ブラウザ DevTools で幅 390px 前後にする。
2. `/logs`、`/logs/new`、`/analytics`、`/machines` を確認する。
3. 長いホール名、長い機種名、マイナス収支でも文字やボタンが重ならないことを確認する。
4. 下部タブと保存ボタンがタップしやすく、フォーム入力を隠し続けないことを確認する。

## 既知の未対応事項

- クラウド同期、ログイン、CSV 入出力は MVP 後。
- 機種 seed は最小データで、外部更新や詳細スペック管理は未対応。
- E2E テストは未導入。MVP では Vitest と手動確認で担保する。
- ホール名の表記ゆれ統合は未対応。
