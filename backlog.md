# Backlog - パチンコ収支管理アプリ MVP

## 目的

ADR 0001/0002 と `docs/screen/layout.pen` の画面案をもとに、スマホで素早く実戦ログを記録し、端末内保存で収支傾向を確認できる MVP を段階的に実装する。

## 前提

- アプリは `Vite + React + TypeScript` のローカルファースト SPA とする。
- データ保存は IndexedDB とし、アクセスには `Dexie.js` を使う。
- 画面遷移は `React Router`、状態取得/更新は `TanStack Query` を使う。
- UI は `Tailwind CSS + shadcn/ui + lucide-react` をベースにする。
- MVP ではバックエンド、ログイン、クラウド同期、E2E テストは対象外とする。
- MVP の初期公開は GitHub Pages とし、個人データは端末内保存のままとする。

### PWT-035: CI で依存関係と秘密情報の安全性を確認する

**ユーザーストーリー**: 開発者として、脆弱な依存関係や公開してはいけない値を含んだまま公開したくない。

**実装メモ**:
- `quality` job に `npm audit --audit-level=high` を追加する。
- `high` 以上の脆弱性がある場合は CI を失敗させる。
- `.env`、`.env.*`、秘密鍵ファイル、AWS キー、GitHub トークン、`SECRET`、`PASSWORD`、`TOKEN=` などを検出する軽量な自作チェックを追加する。
- `.env.example` や説明用ドキュメントは許容する。
- 誤検知や検知漏れが問題になったら、後続で `gitleaks` など専用ツールへの移行を検討する。

**受け入れ条件**:
- [x] `npm audit --audit-level=high` が `quality` job で実行される。
- [x] high/critical の脆弱性がある場合に CI が失敗する。
- [x] `.env` または `.env.*` がコミット対象に含まれる場合に CI が失敗する。
- [x] 秘密鍵ファイルや代表的なトークン文字列が検出された場合に CI が失敗する。

### PWT-036: 公式 Pages Actions で GitHub Pages へデプロイする

**ユーザーストーリー**: 開発者として、ビルド成果物を標準的な方法で GitHub Pages に公開したい。
